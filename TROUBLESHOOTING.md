# Решение проблем с GitHub Pages

## Проблема: Страницы не создаются

Если страницы не создаются на GitHub Pages, проверьте следующее:

### 1. Проверьте конфигурацию

Убедитесь, что в `_config.yml`:
- Убрана тема `minima` (закомментирована)
- Добавлены `include` директории
- Настроены `defaults` для всех папок

### 2. Проверьте front matter

Каждый markdown файл должен иметь:
```yaml
---
layout: default
title: Название страницы
permalink: /путь/к/файлу.html
---
```

### 3. Проверьте структуру файлов

Убедитесь, что:
- Все файлы имеют расширение `.md`
- Все файлы находятся в правильных папках
- Имена файлов на английском (без кириллицы)

### 4. Проверьте настройки GitHub Pages

1. Перейдите в Settings → Pages
2. Убедитесь, что выбрана ветка `main` (или `master`)
3. Папка должна быть `/ (root)`
4. Источник может быть `Deploy from a branch` или `GitHub Actions`

### 5. Проверьте логи сборки

1. Перейдите в раздел **Actions** в репозитории
2. Найдите последний workflow run
3. Проверьте логи на наличие ошибок

### 6. Локальная проверка

Запустите локально для проверки:
```bash
bundle install
bundle exec jekyll serve
```

Откройте http://localhost:4000 и проверьте, работают ли ссылки.

### 7. Очистка кеша

Если изменения не применяются:
1. В Settings → Pages нажмите "Clear cache"
2. Или сделайте новый коммит с небольшим изменением

### 8. Проверка URL

После публикации URL должен быть:
```
https://ваш-username.github.io/репозиторий/00_introduction/01_goals_and_objectives.html
```

Если репозиторий называется `petro_ml_book`, то:
```
https://ваш-username.github.io/petro_ml_book/00_introduction/01_goals_and_objectives.html
```

## Частые ошибки

### Ошибка: "There isn't a GitHub Pages site here"

**Причины:**
- Репозиторий приватный (нужен публичный для бесплатного GitHub Pages)
- GitHub Pages не включен в настройках
- Ошибка в конфигурации Jekyll

**Решение:**
1. Сделайте репозиторий публичным
2. Включите GitHub Pages в Settings → Pages
3. Проверьте `_config.yml` на ошибки

### Ошибка: "Page build failed"

**Причины:**
- Синтаксическая ошибка в `_config.yml`
- Проблемы с front matter в markdown файлах
- Несовместимые плагины

**Решение:**
1. Проверьте логи в Actions
2. Исправьте ошибки в конфигурации
3. Убедитесь, что все плагины поддерживаются GitHub Pages

### Страницы создаются, но не отображаются

**Причины:**
- Неправильные permalinks
- Проблемы с layout
- CSS не загружается

**Решение:**
1. Проверьте permalinks в front matter
2. Убедитесь, что `_layouts/default.html` существует
3. Проверьте пути к CSS в layout

## Полезные команды

```bash
# Локальная сборка
bundle exec jekyll build

# Локальный сервер
bundle exec jekyll serve

# Проверка конфигурации
bundle exec jekyll doctor
```

## Контакты

Если проблема не решена, создайте Issue в репозитории с описанием проблемы и логами из Actions.
